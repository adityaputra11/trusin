SELF-HOSTED WEBHOOK INFRASTRUCTURE

Keep webhook delivery on infrastructure you control.

trusin is open-source webhook delivery infrastructure for teams that want durable event history, routing, retries, and visibility without handing over their operational data plane.

Deploy trusinView source on GitHub

What you run

  • Postgres for organizations, configuration, events, attempts, and audit history.
  • Redis for delivery queues, retry scheduling, and short-lived operational state.
  • trusin services for the backend API, worker, dashboard, CLI, and MCP integration.

Why self-host

Keep provider payloads in your environment, tune infrastructure to your needs, and run the same operational model across development and production. trusin is licensed under Apache-2.0.

Start small, scale deliberately.

Use Docker Compose for local evaluation, then follow the deployment guide for managed Postgres, Redis, HTTPS, and service supervision.
