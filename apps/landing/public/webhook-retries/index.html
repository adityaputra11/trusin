Failures happen. Lost events should not.

trusin records failed delivery attempts and retries retryable failures with exponential backoff, so your team can see exactly what happened and recover deliberately.

Retry with context

A retry is not just a counter. Every attempt retains the response status, latency, response body, and timestamp. This gives operators the context to distinguish a temporary outage from a payload or configuration issue.

  • Exponential backoff avoids overwhelming an unhealthy target.
  • Delivery timelines show every retry in chronological order.
  • Manual retry lets an authorized operator replay a failed event after fixing the target.
Design target endpoints for duplicates.

HTTP timeouts can leave delivery status unknown. Use an event identifier or idempotency key so repeated webhook deliveries do not repeat business actions.
